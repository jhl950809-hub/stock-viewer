# 일별 갱신 사용법

## 1. 수동 실행 (한 번에 전부)

폴더에서 `daily_run.bat` 더블클릭 — 이 한 번이 다음을 순서대로 실행:

1. `update_daily.py` — yfinance에서 빠진 날짜만 받아 기존 parquet에 추가
2. `multi_horizon.py` — 60/120/240일 분류기·회귀기 재학습 (캐시 자동 무효화)
3. `make_forecast_charts.py` — 종목별 예측 차트 재생성
4. `make_html.py` — HTML 대시보드 갱신

소요 시간: 데이터 1일치만 받으면 보통 **3~5분**, 큰 갱신이면 **30분 내외**.

## 2. 시세만 빠르게 갱신

```cmd
py update_daily.py
```

- 종목별 마지막 날짜 자동 감지
- 부족한 부분만 받아서 append (중복 제거)
- 새로 상장된 종목은 자동으로 2010년부터 받기
- 학습/차트는 갱신 안 함 — 차트나 예측이 옛 상태로 남음

## 3. 매일 자동 실행 (Windows 작업 스케줄러)

### Windows 작업 스케줄러 등록 단계

1. 시작 메뉴 → **작업 스케줄러** 검색 후 실행
2. 우측 **작업 만들기** 클릭
3. **일반 탭**: 이름 = "한국주식 일별갱신"
4. **트리거 탭** → 새로 만들기 → 매일, 오후 7:00 (또는 원하는 시간)
   - KRX 정규장 마감(15:30) + 데이터 반영 여유 시간으로 19시 추천
5. **동작 탭** → 새로 만들기 →
   - 동작: "프로그램 시작"
   - 프로그램/스크립트: `cmd.exe`
   - 인수 추가: `/c "C:\Users\정하림\Desktop\새 폴더 (2)\daily_run.bat" >> "C:\Users\정하림\Desktop\새 폴더 (2)\daily.log" 2>&1`
   - 시작 위치: `C:\Users\정하림\Desktop\새 폴더 (2)`
6. **조건 탭**: "AC 전원에서만 실행"이 체크돼 있으면 노트북은 충전 중일 때만 동작 — 데스크탑은 무관
7. 확인 → 비밀번호 입력

### 동작 확인
- 작업 스케줄러에서 해당 작업 우클릭 → **실행** 으로 즉시 테스트
- `daily.log` 파일에 출력이 누적됨

### 주의
- PC가 켜져 있어야 함 (또는 절전모드 깨움 옵션 사용)
- 주말은 KRX 휴장이라 새 데이터 없음 — 그래도 실행은 되며 "이미 최신" 메시지 출력

## 4. 빠진 날짜만 채우기 (이전 누락 보정)

`update_daily.py`는 자동으로 모든 누락 구간을 채웁니다 — 며칠 PC를 안 켰어도 다음 실행 시 한 번에 다 받아옵니다.

특정 종목만 강제로 다시 받고 싶으면 해당 parquet 삭제 후 실행:

```cmd
del "data\ohlcv\005930.parquet"
py update_daily.py
```

→ 삭제된 종목은 2010-01-01부터 전체 다운로드.

## 5. 출력 확인

| 파일 | 용도 |
|---|---|
| `data/update_log.json` | 마지막 갱신 결과 (그룹별 추가 행수, 실패 종목) |
| `daily.log` | 스케줄러 실행 누적 로그 |
| `data/predictions_multi.xlsx` | 갱신된 예측표 |
| `viewer.html` | 갱신된 대시보드 |

## 6. 실패 종목 대응

- **상장폐지/거래정지** 종목은 yfinance에서 데이터를 받지 못해 `update_log.json`의 `failed_tickers`에 누적됨 → 무시해도 됨
- **새로 상장된 종목**은 자동으로 인식되어 처음부터 받기 때문에 별도 작업 불필요

## 7. 의존성 체크
